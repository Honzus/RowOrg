# Deploying RowOrg to Render (free)

This repo ships a `render.yaml` Blueprint that provisions three services on a single Render account:

| Service | What it is | URL |
|---|---|---|
| `roworg-db` | Free PostgreSQL | (internal) |
| `roworg-api` | Django web service | `https://roworg-api.onrender.com` |
| `roworg-frontend` | Static site (Vite build) | `https://roworg-frontend.onrender.com` |

Names get a random suffix if taken — Render will tell you the actual URLs after the first deploy.

## 1. One-time setup

1. Push this repo to GitHub (already done).
2. Sign up at **https://render.com** (use the "Continue with GitHub" option).
3. From the dashboard, click **New +** → **Blueprint**.
4. Pick the `Honzus/RowOrg` repo. Render finds `render.yaml` automatically.
5. Confirm. Render will create the database, API service, and static site, and start the first deploy.

## 2. Wire the frontend to the API

The frontend needs `VITE_API_URL` baked in at build time. It's defined in `render.yaml` as `sync: false`, meaning Render asks for the value the first time. After the first deploy:

1. Note your API's URL from the Render dashboard (something like `https://roworg-api-xxxx.onrender.com`).
2. Open the **roworg-frontend** service → **Environment** tab.
3. Set `VITE_API_URL` to `https://roworg-api-xxxx.onrender.com/api` (mind the `/api` suffix).
4. Click **Manual Deploy** → **Clear build cache & deploy**. The new build will pick up the env var.

## 3. Seed demo accounts (optional)

To get demo users on the deployed instance:

1. Open the **roworg-api** service → **Shell** tab.
2. Run: `python manage.py seed_demo`

Logins: `coach@demo.com`, `alice@demo.com`, etc. — password `row123456` for all.

## 4. Inviting your team

- Log in as `coach@demo.com`, copy the team invite code shown on the **Roster** page.
- Send your teammates the frontend URL + the invite code.
- They register, paste the code on the team setup screen, fill out their profile, and they're in.

## Notes & caveats

- **Free tier sleeps**: the API spins down after 15 minutes of inactivity. First request after that takes ~30s while it cold-starts. The frontend stays up.
- **Free Postgres**: persists indefinitely but Render's free DB tier currently caps at 1 GB and 30 days inactivity grace. For long-term hosting, swap `DATABASE_URL` for a Neon (neon.tech) or Supabase free Postgres — just paste its connection string into the `roworg-api` env vars.
- **CORS**: `CORS_ALLOW_ALL=True` is set for simplicity. Once your URLs are stable, lock it down with `CORS_ALLOWED_ORIGINS=https://your-frontend.onrender.com` instead.
- **SECRET_KEY** is auto-generated by Render on first provision and stored in the service env vars. Don't commit it.
